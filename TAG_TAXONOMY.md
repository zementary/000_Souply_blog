# 标签分类体系 (Tag Taxonomy)

## 设计原则

### ✅ 标签应该做到：
1. **可复用**: 多个视频可以共享同一个标签
2. **分类明确**: 每个标签代表一个可识别的风格/特征
3. **便于发现**: 用户点击标签能找到相似视频
4. **数量可控**: 标签库应该是有限且可管理的（50-200个）

### ❌ 避免：
1. ~~为每个视频创造独特标签~~
2. ~~过于具体的描述性标签~~
3. ~~标签变成视频摘要~~

## 标签使用指南

### 每个视频应该有多少个标签？
- **推荐**: 3-6 个标签
- **最少**: 2 个
- **最多**: 8 个

### 标签选择优先级
1. **核心视觉特征** (必选 1-2 个)
2. **技术手法** (可选 1-2 个)
3. **主题/情感** (可选 1-2 个)
4. **场景/环境** (可选 0-1 个)

### 标签组合示例

**示例 1: 舞蹈类 MV**
```yaml
tags: ["dance-choreography", "synchronized", "one-take", "urban"]
```

**示例 2: 概念性 MV**
```yaml
tags: ["surreal", "vfx-heavy", "abstract", "narrative"]
```

**示例 3: 表演类 MV**
```yaml
tags: ["solo-performance", "black-and-white", "minimal", "studio"]
```

## 标签维护

### 定期审查
1. 统计标签使用频率
2. 合并相似标签
3. 删除使用率低的标签（< 2 次使用）
4. 识别缺失的常用标签

### 标签库增长
- 新标签必须至少适用于 2 个以上的视频
- 新标签需要清晰定义和示例
- 避免创建过于具体的标签

## CSV 标签填写指南

### 旧的方式 ❌
```csv
Visual_Hook
Era-Defining Internet Panopticon  # 太具体，无法复用
Manic Spitting Montage           # 太具体，无法复用
```

### 新的方式 ✅
```csv
Tags
dance-choreography,meta,crowd-scene,high-energy
rapid-editing,black-and-white,surreal,performance
surreal,narrative,single-location,dystopian
```

或者在 CSV 中保留 Visual_Hook 作为描述，但在代码中映射到标准标签：

```csv
Visual_Hook,Suggested_Tags
Era-Defining Internet Panopticon,dance-choreography;meta;crowd-scene
Manic Spitting Montage,rapid-editing;black-and-white;surreal
```
